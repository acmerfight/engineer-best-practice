# 2026 工程技术选型参考

基于事实的云服务与技术栈选型分析，面向开发者。

## 原则

- **严格基于事实**：每个结论标注信息来源（官方文档 / 公开状态页 / 生产案例 / 营销材料），区分强证据与弱证据。
- **标注偏见**：当信息来源方存在利益关联时明确标注。
- **写明局限**：每项技术的适用边界和已知缺陷与适用场景篇幅对等。
- **不做推荐，做分析**：提供决策所需的事实，不替读者做选择。

## 涵盖范围

每项技术分析包含：

1. **解决什么问题** — 问题定义，为什么需要这类服务
2. **如何解决** — 架构设计，关键技术机制
3. **前端/后端使用方式** — 最小可用代码示例
4. **关键指标** — 性能、可用性、规模数据及其来源
5. **适用与不适用场景** — 明确边界
6. **诚实的局限** — 已知缺陷、edge case、自报数据的可信度
7. **定价** — 免费层和计费模型
8. **生产案例** — 有据可查的真实部署

## 信息时效

内容基于 2026 年 5 月可获取的公开信息。云服务变化快，使用前请验证关键信息是否仍然准确。
