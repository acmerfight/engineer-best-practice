# 2026 工程最佳实践

2026 年开发时推荐的云服务与技术栈。每个领域选出我认为最好的选择，给出依据。

## 原则

- **有立场**：每个领域给出明确推荐，不搞"各有优劣自己选"。
- **依据充分**：推荐基于事实（官方文档 / 公开数据 / 生产案例），标注来源，区分强证据与弱证据。
- **诚实标注局限**：推荐的技术也写清楚已知缺陷和不适用场景。

## 目录

| 主题 | 文件 | 状态 |
|------|------|------|
| 实时数据推送基础设施（SSE / WebSocket / Pub-Sub） | [realtime-push-infrastructure.md](realtime-push-infrastructure.md) | **推荐 Ably**。备选方案（Cloudflare DO / SSE 自建 / Centrifugo）待补充 |

## 每项推荐包含

1. **为什么选它** — 核心推荐理由
2. **解决什么问题** — 问题定义
3. **如何解决** — 架构设计与关键技术机制
4. **怎么用** — 最小可用代码示例
5. **关键指标** — 性能、可用性数据及来源
6. **局限** — 已知缺陷和不适用场景
7. **定价与生产案例**

## 信息时效

内容基于 2026 年 5 月可获取的公开信息。云服务变化快，使用前请验证关键信息是否仍然准确。
